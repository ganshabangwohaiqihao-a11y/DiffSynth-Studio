## 2. 这个脚本评什么

当前支持的指标有：

- `IoU`
- `CNT-2D`
- `OAR-2D`
- `NAV-2D`
- `2D OOB / mask-respect`
- `semantic CKL`
- `FID / KID`
- `SCA`

这些指标都只评估：

- `SLDN` 输出的 `2D semantic layout`

不评估：

- `APM`
- 检索后的 3D 物体
- 最终 scene-state

---

## 3. 评估需要什么输入

### GT

GT 来自预处理后的 `npy` 文件，例如：

- `datasets/.../chinese_train_128x128.npy`
- `datasets/.../chinese_val_128x128.npy`
- `datasets/.../chinese_test_128x128.npy`

当前默认读取：

- `channel_index = 1`

也就是：

- `(2, 128, 128)` 里的第 2 个通道作为语义图

### Prediction

预测结果通常来自：

- [scripts/inference_npy.py](/c:/Users/紫燕/Desktop/SemLayoutDiff_update_20260312_163509/scripts/inference_npy.py)

最推荐直接用：

- `*_pred.npy`

典型文件名例如：

- `sample_0_no_text_pred.npy`
- `sample_0_with_text_pred.npy`

原因是这些指标：

- `IoU`
- `CNT-2D`
- `OAR-2D`
- `NAV-2D`
- `2D OOB`
- `CKL`

都更适合直接基于 label map 计算。

## 4. 最常用的命令

### 4.1 评估测试集 `no_text`

```powershell
python scripts/evaluate_sldn_layouts.py `
  --config configs/evaluation_sldn_layout.yaml `
  --gt-data-dir datasets/_tmp_parse_floorplan_map_full `
  --split test `
  --pred-dir inference_output/job1133953_few_best/test `
  --pred-glob "*_pred.npy" `
  --pred-index-regex "sample_(\d+)" `
  --name-filter no_text `
  --output-dir inference_output/job1133953_few_best_eval/test_no_text
```

### 4.2 评估测试集 `with_text`

```powershell
python scripts/evaluate_sldn_layouts.py `
  --config configs/evaluation_sldn_layout.yaml `
  --gt-data-dir datasets/_tmp_parse_floorplan_map_full `
  --split test `
  --pred-dir inference_output/job1133953_few_best/test `
  --pred-glob "*_pred.npy" `
  --pred-index-regex "sample_(\d+)" `
  --name-filter with_text `
  --output-dir inference_output/job1133953_few_best_eval/test_with_text
```

### 4.3 评估训练集

把上面的：

- `--split test`

改成：

- `--split train`

再把：

- `--pred-dir`

改成训练集预测目录即可。

### 4.4 只跑部分指标

如果只想跑指定指标，可以加：

```powershell
--metrics iou count_2d oar_2d nav_2d oob_2d ckl
```

例如：

```powershell
python scripts/evaluate_sldn_layouts.py `
  --config configs/evaluation_sldn_layout.yaml `
  --gt-data-dir datasets/_tmp_parse_floorplan_map_full `
  --split test `
  --pred-dir inference_output/job1133953_few_best/test `
  --pred-glob "*_pred.npy" `
  --pred-index-regex "sample_(\d+)" `
  --name-filter no_text `
  --metrics iou count_2d oar_2d nav_2d oob_2d ckl `
  --output-dir inference_output/job1133953_few_best_eval/test_no_text_light
```

## 5. 输出结果会写什么

每次评估目录里最重要的文件有：

- `matched_samples.json`
  - 实际匹配上的样本列表

- `iou_results.json`
  - `pixel_accuracy`
  - `mean_iou_all_present_classes`
  - `mean_iou_furniture_present_classes`
  - `occupancy_iou`

- `count_2d_results.json`
  - `count_instance_f1`
  - `count_class_exact_match_ratio`

- `oar_2d_results.json`
  - `oar_relation_f1`
  - `oar_relation_exact_match_ratio`

- `nav_2d_results.json`
  - `mean_pred_navigability_ratio`
  - `mean_gt_navigability_ratio`
  - `mean_navigability_abs_error`

- `oob_2d_results.json`
  - `furniture_oob_pixel_ratio`
  - `mask_respect_ratio`
  - 以及门窗阻挡相关结果

- `semantic_ckl_results.json`
  - `semantic_ckl`

- `fid_kid_results.json`
  - `fid`
  - `kid`

- `sca_results.json`
  - `sca`

- `evaluation_summary.json`
  - 当前这次评估的总汇总

## 6. 结果优先怎么看

如果是正常比较 checkpoint，建议先看：

### 第一层：paired 质量

- `mean_iou_furniture_present_classes`
- `occupancy_iou`

### 第二层：合理性与约束

- `count_instance_f1`
- `count_class_exact_match_ratio`
- `oar_relation_f1`
- `oar_relation_exact_match_ratio`
- `mean_navigability_abs_error`
- `furniture_oob_pixel_ratio`
- `mask_respect_ratio`

### 第三层：分布一致性

- `semantic_ckl`
- `fid`
- `kid`

### 次级参考

- `sca`
- 可视化

## 7. 最常见的使用场景

### 场景 A：比较 `no_text` 和 `with_text`

做法：

- 同一个预测目录
- 分别用：
  - `--name-filter no_text`
  - `--name-filter with_text`

然后比较两个输出目录的 `evaluation_summary.json`

### 场景 B：比较不同 checkpoint

做法：

- 每个 checkpoint 先各自跑一遍预测
- 再分别跑评估
- 最后把每个目录的 `evaluation_summary.json` 汇总成表

### 场景 C：只做快速调试

做法：

- 限制样本数
- 或只跑：
  - `iou`
  - `count_2d`
  - `oar_2d`
  - `nav_2d`
  - `oob_2d`
  - `ckl`

这样速度会快很多
